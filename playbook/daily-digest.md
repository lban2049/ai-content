# 每日 AI 信息简报 — 执行手册

## 目标

每天自动生成一份 AI 领域信息简报，筛选高价值内容，提交到仓库 `digests/` 目录。作为 inbox 的上游输入，供人工浏览后决定哪些值得转入选题。

## 执行环境

远程 Claude Code Agent（通过 Schedule 定时触发），在云端沙箱中运行。

**可用工具：**
- `opencli`（主力）— 命令行工具，结构化获取各平台数据
- `WebSearch` — 通用搜索补充
- `WebFetch` — 抓取 opencli 未覆盖的网页

## 执行流程

```
采集（各数据源并行）→ 合并去重 → 评分筛选 → 分类整理 → 生成简报 → 提交仓库
```

### Step 1: 数据采集

按数据源清单逐一采集。可将独立的源并行执行以提高效率。每个源的原始输出暂存，进入下一步统一处理。

### Step 2: 合并去重

- **时效性过滤**：只保留 48 小时内发布的内容
- **去重**：相同 URL 只保留一条；标题相似度 > 80% 时保留热度更高的

### Step 3: 评分筛选

为每条内容计算综合分数，用于排序：

**热度权重**（各平台指标）：
- HackerNews: score 值
- Reddit: upvotes
- GitHub: stars today
- arXiv/HuggingFace: 引用数/下载量
- Twitter: likes + retweets

**关键词加分**：

| 优先级 | 关键词 | 加分 |
|--------|-------|------|
| 高 | Claude, GPT-5, Gemini, DeepSeek, o3/o4 | +30 |
| 中 | GPT, LLaMA, Qwen, Mistral, agent, reasoning, multimodal | +20 |
| 低 | diffusion, transformer, fine-tuning, RLHF, RAG | +10 |

**内容创作相关性加分**：与本仓库三大内容栏目（实测对比、工作流拆解、趋势判断）直接相关的 +15。

### Step 4: 分类整理

按分数排序后，分配到 5 个主题分类：

| 分类 | 说明 | 上限 |
|------|------|------|
| 研究 (Research) | 论文、学术成果 | 8 |
| 工具与模型 (Tools & Models) | 开源项目、模型发布、开发工具 | 10 |
| 行业动态 (Industry) | 产品发布、融资、行业分析 | 8 |
| 社区讨论 (Community) | HN/Reddit/X 热门讨论 | 5 |
| 中文资讯 (Chinese) | 中文 AI 新闻 | 5 |

**总输出上限：40 条**（含 5 条精选）。

### Step 5: 生成简报

从所有条目中取 **top 5** 作为精选 Highlights（跨分类），这 5 条不在分类中重复出现。

输出文件：`digests/YYYY-MM-DD.md`

格式模板见下方"输出格式"。

### Step 6: 提交仓库

将生成的简报文件 commit 到仓库 main 分支。Commit message 格式：`Add daily digest for YYYY-MM-DD`

## 数据源清单

> 扩展方式：直接在对应区域追加条目即可。

### 社区热点（高优先级）

| 数据源 | 采集命令 | 筛选规则 |
|--------|---------|---------|
| HackerNews Top | `opencli hackernews top --limit 30` | AI 相关，score > 50 |
| HackerNews Show | `opencli hackernews show --limit 10` | AI 相关项目 |
| Reddit r/MachineLearning | `opencli reddit subreddit MachineLearning --limit 15` | 热帖 |
| Reddit r/LocalLLaMA | `opencli reddit subreddit LocalLLaMA --limit 15` | 热帖 |
| Reddit r/artificial | `opencli reddit subreddit artificial --limit 10` | 热帖 |

## 与现有工作流的衔接

1. **digest → inbox**：每天浏览简报（对应 workflow.md "信息输入 15 min"），将值得深入的条目手动追加到 `inbox/index.md`
2. **inbox → topics**：周一 review inbox 时从中选题
3. **追加原则**：digest 文件生成后不修改，每天一份独立文件

## 扩展指南

### 添加 X/Twitter 用户
在"X/Twitter 关注用户"表格追加一行：
```
| @username | `opencli twitter search "from:username" --limit 5` | 描述 |
```

### 添加 Reddit 频道
在"社区热点"表格追加：
```
| Reddit r/subreddit | `opencli reddit subreddit subreddit --limit 10` | 筛选规则 |
```

### 添加 RSS/Newsletter
在"Newsletter / 博客"表格追加：
```
| 名称 | WebFetch `https://feed-url/rss.xml` | 描述 |
```
或 Substack：
```
| 名称 | `opencli substack publication <publication-url>` | 描述 |
```

### 添加新平台
运行 `opencli list` 查看所有可用平台命令，然后在对应分类追加条目。

### 修改关键词偏好
编辑 Step 3 中的"关键词加分"表格，调整关注领域的权重。
