# Platform Rules

## 公众号

### 基本参数
- 频率：每周 1 篇
- 字数：2000-4000 字
- 发布日：建议周六（工作日积累，周末发布）

### 格式要求
- 有清晰的标题层级（H2/H3）
- 善用对比表格
- 配截图和实际操作画面
- 开头直接给结论，不铺垫
- 结尾给出明确的行动建议（"如果你是 XX 场景，推荐用 YY"）

### 调性
- 系统化、有结论、可复用
- 专业但不学术，像一个靠谱同事在跟你分享
- 允许说"我不确定"和"这个我还没验证"

## X (Twitter)

### 基本参数
- 频率：每天 1-3 条
- 形式：单条短观察 或 thread

### 内容类型
- 使用中的即时感受（"刚发现 Claude Code 的 XX 功能比 Codex 好用在..."）
- 工具速评（"试了 XX 半天，一句话总结..."）
- 实践碎片（"今天用 AI 做了 XX，节省了 N 小时"）
- 对他人观点的回应（附带自己的实践验证）

### 调性
- 随手记录、快速反应、真实
- 可以不完整，可以是半成品想法
- 鼓励互动和讨论

### X → 公众号的转化
日常 X 上发的碎片观察，积累到一定程度整合为公众号长文。长文素材自然积累，不用每周从零写起。
