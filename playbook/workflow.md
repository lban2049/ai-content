# Workflow

## 每日节奏（~1 小时）

### 信息输入（15 min）
- 查看 `digests/` 中当天的自动简报（执行手册见 `playbook/daily-digest.md`）
- 只看 AI 筛选后的摘要，标记值得关注的
- 有价值的素材追加到 `inbox/index.md`

### X 输出（10 min）
- 基于当天使用体感或标记内容，发 1-3 条
- 不用精修，实践者的随手记录就是最好的格式

### 公众号推进（30 min）
- 周一：选题确定 + 大纲
- 周二-周四：分块写作（AI 辅助初稿，自己改观点和结论）
- 周五：终稿校验
- 周六：发布

### 素材归档（5 min）
- 当天的观察、截图、想法归入 inbox 或对应 topic

## 每周节奏

- **周一**：review inbox → 确定本周公众号选题 → 建 topic → 写 brief
- **周二-周四**：推进草稿，积累素材
- **周五**：完成终稿
- **周六**：发布公众号
- **周日**：rest / 轻量素材收集

## 每月节奏

- 月底在 `playbook/review-log.md` 追加复盘
- 复盘内容：哪类内容表现好、受众增长、下月调整方向
- 清理 inbox 中过期素材

## AI 自动化触点

| 环节 | AI 负责 | 人负责 |
| --- | --- | --- |
| 素材收集 | 定时 Agent 执行 daily-digest 采集简报，提交到 `digests/` | 浏览简报，判断哪些值得深入，追加到 inbox |
| 选题 | 根据栏目和定位推荐选题优先级 | 最终选题决策 |
| Brief | 根据素材生成 brief 初稿 | 确认核心观点和差异化 |
| 大纲 | 生成结构化大纲 | 调整逻辑和重点 |
| 初稿 | 基于大纲和素材生成初稿 | 注入真实体验、改文风、加判断 |
| 格式转换 | 公众号文章 → X thread | 最终 review |
| 数据复盘 | 汇总阅读/互动数据 | 写复盘笔记 |
