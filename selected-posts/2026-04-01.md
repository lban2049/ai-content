> Carl Vellotti ran a research task in Claude Code. Ten tool calls. Almost 30,000 tokens. Web searches, page reads, synthesis.

When it finishes, only the summary returns to your main session. The 30,000 tokens of raw work never touch your primary context.
>
> Without the sub-agent, that same task would have pushed context to 25%. Five messages in and you're waiting three minutes for compaction. With the sub-agent, the main session barely noticed.
>
> This is the part that changes how you plan your sessions. Context is finite. Every file Claude reads, every web page it scrapes, every tool call it runs eats into that window. Most people hit compaction because they're running everything in a single thread. Sub-agents let you run arbitrarily complex tasks while keeping your main session clean for the decisions that actually need your full context history.
>
> Two ways to set them up. Define dedicated sub-agents in advance for tasks you repeat. Or let Claude spin one up on the fly for any task. Both work. The on-the-fly approach requires zero setup.
