> "A perfect score is unlikely for any current agent." — MongoDB's Developer Education team, launching the Agent Reading Test.

> Claude Opus scored 19/20 with a simpe change.

> 1. What the test measures
>
> The Agent Reading Test isn't a vibe check. 10 tests, 20 points. Each test targets a specific failure mode in how agents read the web:
>
> - Truncation at 130,000 characters
> - Content buried under 80,000 characters of CSS and JS
> - JavaScript-only SPA rendering
> - Tabbed content across 8 languages
> - Soft 404 detection
> - Broken markdown fence recovery
> - Content negotiation (serve clean markdown when asked)
> - Cross-host redirects
> - Generic headers hiding platform-specific content
> - Navigation chrome before real documentation
>
> 16 canary tokens + 4 qualitative checks. Max 20 points.
>
> 2. Default WebFetch: 7/20
>
> Claude Code ships with a built-in WebFetch tool. For simple pages, it's adequate. Against this benchmark, it failed everything that requires real rendering or header negotiation.
>
> SPA rendering: miss. WebFetch doesn't execute JavaScript. A React app returns its HTML shell — which is empty.
>
> Content buried under 80K of CSS and JS: miss. The tool fetches what the server sends before rendering. If meaningful content is deep in the DOM, it's not there.
>
> Content negotiation: miss. WebFetch sends default headers. A server that will serve clean markdown if asked doesn't know to serve it.
>
> Seven points is what you get when your agent fetches the same way a 2010 crawler did.
>
> Same Claude Opus. 12-point swing.
>
> 3. Swapping in agent-browser
>
> Vercel Labs built agent-browser as an open-source browser automation CLI designed specifically for agents. Real browser, real rendering, real JavaScript execution. It strips navigation chrome, handles cross-host redirects, waits for rendering, and detects soft 404s.
>
> npm install -g agent-browser
>
> That's the entire change.
>
> 4. 19/20. Every test except one.
>
> SPA rendering: passed. JavaScript executes, DOM renders, actual content returns.
> Content under 80K of CSS and JS: passed.
> Cross-host redirects: passed.
> Navigation chrome: passed. Agent-browser returns content, not layout scaffolding.
> Soft 404 detection: passed.
> Tabbed content across 8 languages: passed.
> Truncation at 130K chars: passed.
>
> The miss: content negotiation. Serving clean markdown requires sending Accept: text/markdown in the request. Agent-browser doesn't add this header by default. One point left on the table — and a solvable one.
>
> 5. What this means for your agents
>
> People optimize prompts. They try different models. They tune system instructions.
>
> The agent's ability to read the web is not a model problem. It's a fetch pipeline problem.
>
> Claude Code with WebFetch and Claude Code with agent-browser are the same Claude model. The 12-point gap is entirely in the layer between the model and the web.
>
> Most agents failing on web content are failing on fetch, not reasoning.
>
> You're seeing fetch errors that look like reasoning errors.
