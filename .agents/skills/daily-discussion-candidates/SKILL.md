# Daily Discussion Candidates

这个 skill 用于本项目内的人工日常选题讨论。

目标不是总结新闻，而是从最近收集的素材中提炼出 3-5 个“值得继续讨论、可以补验证、可能进入发帖流程”的方向。

## 读取范围

优先读取最近 1-3 天的以下目录：

- `digests/`
- `selected-posts/`

必要时参考以下目录，避免重复选题：

- `posts/`
- `topics/`
- `inbox/`

不要批量扫描整个仓库。先读最新文件，不够再向前补一天。

## 工作原则

- 先判断是否有实践支撑，再判断是否值得写
- 不把新闻事件直接当成选题
- 不输出“AI 很强”“这个更新值得关注”这类空判断
- 优先选择和用户真实工作流、工具使用、内容生产方式有关的方向
- 如果素材不足以支撑观点，明确指出还缺什么验证，不要强行立题

## 选择标准

优先保留满足至少两项的方向：

- 能形成一个明确、可被质疑的判断
- 和用户已有实践有连接点
- 可以在 1-2 天内补完验证
- 和最近已发布内容不重复
- 适合先讨论，再进入 `idea-to-post`

主动降权以下素材：

- 纯新闻通稿
- 只有热度，没有方法论外延
- 必须依赖外部二手资料才能成立的判断
- 和本项目内容定位不一致的泛行业资讯

## 输出要求

只输出 3-5 个候选方向，按优先级排序。

每个候选方向必须包含：

- 标题
- 核心判断
- 对应素材
- 为什么值得用户写
- 当前实践支撑
- 还缺什么验证
- 建议下一步
- 适合形式：`X`、`公众号` 或 `待定`

具体格式见 `references/output-format.md`。

## 讨论姿态

- 用“实践者视角”说话，不用营销语气
- 先给判断，再给理由
- 如果方向还不够成熟，直接说“不建议写”
- 可以建议合并相近素材，形成一个更强的角度

## 边界

这个 skill 到“给出讨论候选方向”就结束。

不要：

- 直接生成最终帖子
- 擅自创建 topic
- 把候选方向自动追加到 `inbox/index.md`
- 跳过“还缺什么验证”这一步
